# 🚀 Деплой v3 — фронт + бэкенд с базой данных

Эта версия отличается от предыдущей: данные теперь хранятся не в Telegram CloudStorage,
а в полноценной базе PostgreSQL, привязанной к твоему Telegram user_id. Это значит:
данные не пропадут даже если что-то случится с ботом или CloudStorage Telegram.

---

## Что изменилось

- **Новые разделы:** Вес и калории, Отчёт за месяц, заглушка ИИ-агента
- **Стандартизированные названия упражнений** (вместо "Блок к поясу" → "Тяга нижнего блока к поясу")
- **Offline-first режим** — действия сохраняются на устройстве и отправляются после возвращения сети
- **Свои препараты** — можно добавлять и удалять
- **Даты добавок** теперь генерируются динамически на год вперёд, не зависят от даты сборки
- **Бэкенд на FastAPI + PostgreSQL** — данные привязаны к Telegram user_id
- **Кнопка ручного экспорта в JSON** — на вкладке ИИ-агента

## ШАГ 1 — База данных (Supabase, бесплатно)

1. Зайди на **supabase.com** → Sign up → New Project
2. Введи название проекта, придумай пароль для базы (сохрани его)
3. Дождись создания проекта (~2 минуты)
4. Зайди в **Project Settings → Database**
5. Скопируй **Connection string** (вид URI) — это и есть `DATABASE_URL`

Выглядит примерно так:
```
postgresql://postgres:[ТВОЙ-ПАРОЛЬ]@db.xxxxxxxxxxxx.supabase.co:5432/postgres
```

> Таблицы создавать вручную не нужно — бэкенд создаст их сам при первом запуске.

---

## ШАГ 2 — Бэкенд (Railway, бесплатный тариф)

1. Зайди на **railway.app** → Login через GitHub
2. Создай новый репозиторий на GitHub с папкой `backend/` (файлы: `main.py`, `requirements.txt`, `Procfile`, `railway.json`)
3. В Railway: **New Project → Deploy from GitHub repo** → выбери репозиторий
4. Зайди в настройки проекта → **Variables** → добавь:
   ```
   DATABASE_URL = postgresql://postgres:...  (из Шага 1)
   BOT_TOKEN = токен Telegram-бота
   ALLOWED_ORIGINS = https://planner-frontend-sable.vercel.app
   ALLOW_INSECURE_DEMO = false
   ```
5. Railway автоматически задеплоит — дождись статуса **Active**
6. В разделе **Settings → Networking** нажми **Generate Domain** — получишь URL вида:
   ```
   https://your-project-production.up.railway.app
   ```
7. Проверь что работает — открой этот URL в браузере, должно показать:
   ```json
   {"status":"ok","service":"planner-api"}
   ```

---

## ШАГ 3 — Подключить фронт к бэкенду

1. Открой `index.html` в текстовом редакторе
2. Найди строку (примерно в начале `<script>`):
   ```js
   const API_BASE = 'https://YOUR-BACKEND.up.railway.app';
   ```
3. Замени на свой реальный URL из Шага 2 (без слэша в конце)
4. Сохрани файл

---

## ШАГ 4 — Деплой фронта на Vercel (как раньше)

1. Залей обновлённые `index.html` и `sw.js` в свой репозиторий на GitHub
2. Vercel передеплоит автоматически за ~30 секунд
3. Открой бота в Telegram → кнопка меню → приложение должно открыться с твоей историей тренировок

---

## Как проверить что всё работает

1. Открой приложение → вкладка **🤖 ИИ** (внизу справа)
2. Раздел "Данные и синхронизация" — должен показывать:
   - 🟢 зелёная точка + "Подключено к серверу" — всё работает
   - 🟠 оранжевая точка + "Офлайн-режим" — бэкенд не отвечает, проверь `API_BASE` и переменную `DATABASE_URL` в Railway

3. Открой вкладку **🏋️ Тренировка** → переключись на "Спина+Би" — должна быть видна твоя становая тяга, если зайти на дату из истории через экран Прогресс

---

## Важно про офлайн-режим

Если бэкенд недоступен (не настроен ещё, упал, или нет интернета) — приложение **не ломается**.
Оно открывается из Service Worker-кеша, сохраняет изменения локально и ставит их в очередь.
При возвращении сети операции отправляются на сервер по порядку. Сервер не перезаписывает
локальную копию, пока очередь не завершена.

---

## Резервное копирование

На вкладке **🤖 ИИ** есть кнопка **"⬇️ Скачать резервную копию (JSON)"** — сохраняет все твои данные
(тренировки, задачи, добавки, вес, калории) в файл на телефон/компьютер. Делай это раз в месяц
для дополнительной подстраховки, даже с работающей базой данных.

## Структура файлов

```
твой-репозиторий-фронт/
├── index.html          ← деплоится на Vercel
└── sw.js               ← офлайн-кеш приложения

твой-репозиторий-бэкенд/
├── main.py              ← FastAPI приложение
├── requirements.txt      ← Python-зависимости
├── Procfile              ← команда запуска для Railway
└── railway.json          ← конфиг деплоя
```

Это **два отдельных репозитория** — один для фронта (Vercel), один для бэкенда (Railway).

## Частые проблемы

**Бэкенд показывает Internal Server Error**
→ Проверь что `DATABASE_URL` в Railway указан верно, без лишних пробелов

**Фронт не видит бэкенд (всегда офлайн-режим)**
→ Проверь `API_BASE` в `index.html` — должен быть без слэша на конце, начинаться с `https://`

**CORS ошибка в консоли браузера**
→ Проверь `ALLOWED_ORIGINS`: там должен быть точный origin фронтенда без пути и завершающего `/`

**Данные не появляются на новом устройстве**
→ Убедись что используешь того же Telegram-аккаунта — данные привязаны к `user_id`, не к устройству
